## What are Semantic Tags?

Semantic tags **clearly describe the meaning and role of content to** browsers, search engines, and screen readers.

They explain **what the content is**, not **how it looks**.

## Why Semantic Tags Exist

Semantic tags exist to:

- Improve **accessibility** by helping disabled users use the website easily
- Improve **SEO** so search engines understand the page better
- Create a **clear and meaningful page structure**
- Make code **easy to read and maintain**

## Common Semantic HTML Tags (Clear Meaning)

- `<header>`  
  Used for **top content of a page or section**  
  Usually contains: logo, title, short intro, or actions

- `<nav>`  
  Used **only for navigation links**  
  Example: main menu, sidebar menu, footer links

- `<main>`  
  Contains the **main unique content of the page**  
  Only **one `<main>` per page**

- `<section>`  
  Used to **split a page into meaningful parts**  
  Example: features section, services section

- `<article>`  
  Used for **independent content that can stand alone**  
  Example: blog post, news item, product card

- `<aside>`  
  Used for **extra or supporting content**  
  Example: sidebar, ads, related links, tips

- `<footer>`  
  Used for **ending information of a page or section**  
  Example: copyright, author info, related links

> Use `<div>` only when **no semantic meaning fits**.

> Tags like `<header>`, `<nav>`, `<main>`, `<section>` are **layout semantic tags**.

> Tags like `<h1>`, `<p>`, `<button>`, `<a>`, `<form>` are **content semantic tags**.
